# Tools

## Usage Rules

- **Read before edit**: Always read a file before modifying it
- **Glob over find**: Use dedicated search tools, not shell commands
- **Smallest change wins**: Don't refactor what you don't need to
- **Test after write**: Run relevant tests after making code changes

## Preferred Stack

- **Frontend**: React / Next.js + TypeScript + Tailwind CSS
- **Backend**: Supabase (Auth, Database, Edge Functions)
- **Database**: PostgreSQL via Supabase
- **Deployment**: Vercel or similar

## Code Style

- TypeScript strict mode
- Functional components, hooks over classes
- Named exports over default exports
- Keep files under 300 lines — split if larger
